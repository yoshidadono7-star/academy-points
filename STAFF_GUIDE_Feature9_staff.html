1. 5つの画面とアクセス先

HERO'S ACADEMY には 5 つの画面があります。
各画面の URL（青い文字）をクリックすると、直接その画面が開きます。

⚠️ この資料にはログイン情報を含みます。スタッフ限定で、外部への共有は禁止です。
🛰 管理画面（先生・教室長用）
https://hero-s-points.web.app/

システム全体の管理。生徒登録、スケジュール管理、報酬、統計、設定などすべてをここから操作します。

ログイン方式 Firebase Auth（メール + パスワード） メールアドレス admin@heros.jp パスワード 0070heros
👨‍🎓 生徒画面
https://hero-s-points.web.app/student

生徒がログインして学習する画面。バディAI「ルカ」との会話、学習タイマー、ポイント確認、報酬交換など。

ログイン方式 ニックネーム + 4桁 PIN ニックネーム （管理画面で生徒ごとに設定） 生徒 PIN（4桁） （管理画面で設定した4桁の数字）

例: ニックネーム「ジロー」 / PIN「2000」

https://hero-s-points.web.app/parent

お子さまの学習状況を確認する画面。家庭学習の承認ボタン、レポート受信もこちらで。

ログイン方式 ニックネーム + 保護者 PIN ニックネーム （お子さまのニックネーム） 保護者 PIN（4桁） （生徒 PIN とは別の数字）

例: ニックネーム「ジロー」 / 保護者 PIN「5678」

👨‍🏫 講師画面（教室長もここから）
https://hero-s-points.web.app/teacher

講師がスケジュール・担当生徒のコメント記入、AI 相談、申し送りを行う画面。

ログイン方式 4桁 PIN のみ PIN（4桁） （管理画面の講師管理で設定） 役割判定 講師 or 教室長（management 画面で role を設定）

例: 黒木先生 → PIN「1234」。role=director の講師は教室長バディが有効になります

📺 教室モニター
https://hero-s-points.web.app/briefing

教室の大画面に常時表示する画面。今日の講師シフト・ランキング・ハイライトなど。

ログイン方式 なし（URL を開くだけで表示）

2. 今回の目玉: 階層型 AI コーチング

これまで「ルカ」という生徒用バディ AI だけでしたが、今回から講師・教室長・オーナーにも専用の AI バディが付きました。 しかも 4 階層の間で「申し送り（HandoffLog）」が自動的に流れる仕組みになっていて、 たとえば「生徒がバディに家庭の悩みを話した」→ 自動的に教室長の Inbox にアラートが届く、ということが実現できます。

📐 階層構造のイメージ

👑 オーナー — 経営判断・複数教室統括 (AIBuddyOwner: Opus 4.6)
教室長からの申し送りを要約・優先順位付け。経営判断を一緒に考える。
🏛 教室長 — 教室運営・人材管理 (AIBuddyDirector: Sonnet 4.6)
講師配置、生徒の継続率、保護者対応、教室 KPI を一緒に考える。
👨‍🏫 講師 — 生徒への接し方・授業計画 (AIBuddyTeacher: Sonnet 4.6)
担当生徒一人ひとりへの声かけ例、授業の進め方を一緒に考える。
👨‍🎓 生徒 — 学習サポート・感情サポート (AIBuddy ルカ: Haiku 4.5)
学習計画、モチベーション、勉強の質問など。教室長と同じレベルの情報を持つ。
💡 情報の流れ（上下双方向）
生徒がルカに相談 → ルカが「これは教室長に伝えるべき」と判断 → 自動的に申し送り Inbox へ → 教室長が対応
教室長がルカに「〇〇ちゃんに明日この課題を出して」と指示 → ルカが生徒に伝える

🔐 プライバシー配慮: バディ共有フラグ

講師コメントを記入するとき、「🤖 生徒バディ（ルカ）に共有する」というチェックボックスが付きました。

  • チェック ON → ルカが生徒と会話するときに、このコメントを参照できる（例: 「明日までに数学ワーク p.50-60 をやろう」）
  • チェック OFF → ルカには見えない。講師・教室長だけが見る内部メモ（例: 「家庭の事情で集中力低下」）

機微な情報は OFF、学習の指示は ON、という使い分けを推奨します。

3. ルカ（生徒バディ）の進化

これまでルカは生徒の名前・学年・レベル・ポイントしか知りませんでした。 今回から以下の情報を知った上で会話します:

  • 今日の授業予定（スケジュールスロットから）
  • 未完了の宿題・チャレンジ（タスク管理から）
  • 講師からの最近のコメント（buddyVisible=ON のみ、最新3件）
  • 講師からの未読メッセージ
  • 志望校（大学・中学・高校）
  • 苦手科目
  • 連続学習日数（ストリーク）

💬 実際の会話例

生徒「今日、何やればいい？」

ルカ「黒木先生からのお知らせで、数学のワーク50〜60ページをやるように言われてるにゃ📖 まずそこから始めてみてどう？」

生徒「最近どう？」

ルカ「3日連続で頑張ってるにゃ〜！✨ 黒木先生からも「態度good・理解度good」ってコメントもらえてるよ。いい流れが来てる感じ、自分ではどう思う？」

生徒「高校生の物理なんか難しいね」（中3の生徒が発言）

ルカ「ジローはまだ中3だから、今は高校物理は気にしなくて大丈夫ピヨ！まず中学理科をしっかり固めようね♪ 高校行ったら一緒に攻略しようよ！」

※ 学年を考慮して「今やるべきこと」に誘導できるのがポイント。ただの応援ロボから一歩進んで、状況を踏まえた具体的アドバイスができるようになっています。

⚙️ 先生側での操作のコツ

ルカをもっと賢くするには: 管理画面の講師コメント記入時に「🤖 生徒バディに共有する」チェックを ON で保存してください。 それだけでルカがそのコメント内容を踏まえて生徒に話しかけられるようになります。

また、生徒の「苦手科目」「志望校」をちゃんと登録しておくと、ルカの返答が一段と具体的になります。

4. 講師バディ AI NEW

講師画面（/teacher）に新しく 「🤖 AI相談」 タブが追加されました。 担当する生徒一人ひとりについて、「どう接したら？」「次の授業で何やる？」といった相談を AI にできます。

📝 使い方

  1. 講師画面にログイン（4桁 PIN）
  2. 上のタブから「🤖 AI相談」をクリック
  3. 入力欄に相談内容を書く（例: 「〇〇ちゃん、最近やる気なさそうに見えるけどどう接したら？」）
  4. 送信 → AI が担当生徒の最近のコメント・宿題状況・態度・理解度を踏まえて具体的にアドバイス

💡 こんな時に使ってみてください

  • 新人講師で、特定の生徒への接し方に迷った時
  • 次の授業で何をやるか迷った時
  • 保護者にどう伝えるか言い回しを相談したい時
  • 生徒の細かい変化に気づくきっかけを得たい時
💬 AI の特徴
・ 押し付けがましくせず「こういう選択肢がありますよ」と複数案を提示
・ 才能でなく過程を褒める（成長マインドセット）
・ 個人情報配慮。生徒に直接伝えるべきでないことは明示
・ 返答は 200-400 文字程度、箇条書きで読みやすく

5. 教室長バディ AI NEW

同じく講師画面の「🤖 AI相談」タブですが、role=director（教室長）で登録された講師がログインすると、 自動的に 教室長モードに切り替わります。画面上部のヘッダーに「（教室長）」と表示され、 AI のプロンプトも教室運営視点のものに変わります。

📊 教室長モードで取れる相談

  • 講師配置: 「今月シフト減らしたい講師の代わりに誰を増やすべき？」
  • 生徒対応: 「〇〇ちゃんの保護者にどう連絡すべき？」
  • 教室 KPI: 「今月の出席率は？」「継続率で気になる生徒は？」
  • 採用判断: 「講師を増やすべきタイミング？」

👀 教室長が見られる情報

教室長バディは以下の情報を全部見た上で答えます:

  • 在籍生徒数・講師数
  • 今日の全授業（完了・欠席含む）
  • 家庭学習の承認待ち件数
  • 今日の講師コメント（全件）
  • 最近の申し送り（director ロール向け）

⚠️ 講師→教室長の切替方法

既存の講師を教室長に昇格させるには:

  1. 管理画面にログイン
  2. 「👨‍🏫 講師管理」タブ
  3. 該当講師の ✏️ ボタンをクリック
  4. 「役割」を 「教室長」 に変更
  5. 保存

変更後、その講師が次回 /teacher にログインすると、AI 相談タブが自動で教室長モードになります。

6. 申し送り Inbox NEW

講師画面に 「📬 申し送り」 タブが追加されました。階層間で自動・手動に流れる重要情報がここに集まります。 タブ右上に未読件数のバッジが表示されます。

🚨 どんな情報が届くか

アイコン種類内容
🚨alert緊急度高。生徒の心身・家庭・学習に要注意な兆候。即対応推奨
👁observation重要だが緊急ではない気付き。成長・変化の記録
🙏request具体的な要望（「次回〇〇してあげて」等）
✅decision話し合いで決まった方針・約束

🌊 自動で届くケース

生徒がルカに感情的な話をすると、ルカが自動的に申し送りを書き込みます:

生徒「最近、親がケンカしてばっかでしんどい…」

ルカ「お疲れさま、大丈夫？深呼吸してリラックスしよう☕」（生徒画面には優しい返答）

→ 裏側で自動生成:
🚨 alert: 家庭環境にストレス兆候。声かけ推奨。

→ 次に講師・教室長が申し送り Inbox を開くと、このエントリが赤い帯で一番上に表示されます。「既読」ボタンで既読化。

🎯 使い方のコツ

  • 毎朝チェック: 授業前に申し送り Inbox を開いて、前日までの新着を確認
  • 夕会で活用: 教室長が申し送りを元にスタッフ間で情報共有
  • 既読管理: 対応済みは「既読」でバッジから消す

7. 音声入力の常時マイクモード IMPROVED

生徒画面のバディ会話機能で、マイクを 1 回押したらずっと話し続けられるようになりました。

📝 使い方

  1. 生徒画面でチャットバーの 🎤 ボタンをタップ
  2. バディが「🎤 聞いてるよ！話してみて〜」と吹き出しで知らせてくれる
  3. そのまま話す → 自動的に文字化されて送信 → バディが返答
  4. 🎤 を押し直さずにそのまま続けて話せる
  5. 終わりたい時だけ 🎤 をもう一度タップ → 「マイクオフにしたにゃ〜」で停止
✨ これまでとの違い
以前: 1回話す → マイク停止 → 次に話すたびに 🎤 を押し直し（面倒！）
今回: 1回 🎤 → ずっとオン → 好きなだけ連続で話せる → 終わる時だけ 🎤 再タップ

⚠️ ご注意

  • Mac / Windows の Chrome で動作します
  • iPhone / iPad の Chrome では動きません（Safari と同じエンジンを使っていて、音声認識 API 非対応のため）
  • 5 分で自動的にマイクがオフになります（バッテリー・コスト・誤爆防止のため）
  • Mac でクラムシェルモード（ノート閉じ）にしていると、内蔵マイクが使えない場合があります
  • 外部マイク（DJI MIC 等）を USB 接続していると、そちらが優先されます。使いたいマイクが選ばれているか、システム設定 → サウンド → 入力で確認してください

❓ マイクが反応しない時

  1. Mac のシステム設定 → サウンド → 入力 で、使いたいマイクが選ばれているか確認
  2. マイクに向かって話すと「入力レベル」のバーが動くか確認
  3. Chrome のアドレスバー左の🔒アイコン → マイク「許可」になっているか確認
  4. 他のアプリ（Zoom・Teams 等）がマイクを使っていないか確認
  5. それでもダメなら、シークレットウィンドウで開いて試す

8. 艦長マイケルのブリーフィング NEW (Feature 10)

教室モニター（https://hero-s-points.web.app/briefing）の画面上部に、新キャラクター 「艦長マイケル」 が登場しました。宇宙艦隊ミッションコントロールの艦長役として、その日のメインメッセージをクルー（生徒）に向けて一方的に語る情報表示領域です。

🧑‍✈️ 艦長マイケルとは

  • 宇宙艦長風の SVG キャラクター（口髭・艦長帽・金の階級章）
  • 教室モニター画面の上部に常駐し、その日のメインメッセージを吹き出しで表示
  • タイプライター演出 + 回転する通信リング付きで、モニターの存在感 UP
  • 質問・対話機能はなし。一方的にメッセージを語るだけの情報表示

📝 メッセージの仕組み（ハイブリッド方式）

艦長マイケルのセリフは 2 系統のデータソースをハイブリッドで扱います:

  1. AI 自動生成（デフォルト）: 1 日 1 回、Claude API（Haiku 4.5）が以下の情報から艦長セリフを自動生成します。生成結果は Firestore dailyBriefing/{日付} にキャッシュされるので同じ日に何度開いても追加コストはかかりません。
    • 時間帯（朝 / 昼 / 夕方）
    • 今日の出撃中クルー数・総学習時間
    • 今日誕生日の生徒
    • レイドボスの HP 状況
    • 進行中ストーリー
  2. 手動上書き（特別な日だけ）: 管理画面の「🚀 今日のプラン管理」タブ先頭に 「🧑‍✈️ 艦長マイケルの『今日のメッセージ』」 カードがあります。そこに入力して「💾 保存」を押すと、その日のメッセージが塾長が書いた内容で上書きされます（ブリーフィング画面には「手動」タグが表示）。

💡 使い方のコツ

  • 普段は何もしなくて OK。空欄のままなら AI が毎日自動で気の利いたメッセージを生成します。
  • 期末テスト前・大型イベント・進学説明会など「特別な一言を伝えたい日」だけ手動入力。
  • 手動メッセージは「艦長マイケルになりきった口調」で書くと世界観が崩れません（例:「諸君、本日は期末テスト対策最終日だ。全クルー、全力で挑め！」）。
  • 手動メッセージを消して AI 自動に戻すには「🗑️ クリア」ボタン。
  • AI メッセージが気に入らない日は、ブリーフィング画面の艦長横にある「🔄 更新」ボタンで再生成できます。
💰 API コストの目安
1 日 1 回の生成 + キャッシュ方式なので、Claude Haiku のコストで月数十円以下。 手動更新ボタンを連打しない限り、追加料金はほぼ発生しません。

9. スタッフ向け運用メモ

🎯 1 日の流れイメージ

朝（開講前 15 分）

  • 教室長は /teacher にログイン → 📬 申し送り Inbox をチェック
  • 前日までに溜まった未読の alert / observation を確認
  • 当日の担当講師と朝礼で情報共有

授業中

  • 講師は各自の /teacher にログイン → 「📅 今日のスケジュール」で担当を確認
  • 授業終わりに「📝 生徒コメント」から学習態度・理解度・特記事項を記入
  • 特記事項が学習に関することなら 「🤖 バディに共有する」ON で保存
  • 家庭の事情など機微なことは OFF で保存

困った時

  • 「〇〇ちゃんどう接したら？」→ 「🤖 AI相談」タブで相談
  • AI が担当生徒の最近のコメント・態度から具体的なアドバイス
  • 新人講師でも大ベテラン並みの返答ができるように

終礼

  • 教室長は申し送り Inbox を再確認（生徒バディが自動生成したアラート含む）
  • 重要なものを夕会で共有
  • オーナーへの申し送りが必要なものは教室長バディ AI に相談しながらまとめる

💡 豆知識

❓ バディキャラクター名について
「ルカ」「ミミ」「ピヨ」など、生徒によって違うバディキャラが表示されます。これは既存のバディシステムの仕様で、 生徒が選んだキャラクター名が反映されます。内部的にはすべて同じ AI（AIBuddy）が答えています。
❓ プロンプトキャッシングでコスト削減
今回の仕組みでは、AI に毎回送る「共通ガイド部分」をキャッシュすることで、コストが約 45% 削減されています。 月額想定 18,000 円 → 10,000 円。Phase 2-3 追加しても合計約 14,700 円で収まる設計です。
❓ API 過負荷で待たされる時
Anthropic（Claude）のサービスが一時的に混雑すると、返答に 10 秒以上かかることがあります。 この場合、10 秒経過するとバディが「もうちょっとだけ…考え中にゃ…🌀」と吹き出しを出してフリーズを防ぎます。 自動的に最大 3 回までリトライするので、ほとんどの場合は待っていれば返答が返ってきます。

10. トラブルシューティング

画面が古いままで更新されない

症状: 何か新機能が入ったはずなのに、ブラウザで見ても変わってない
原因: Service Worker が古いキャッシュを返している可能性
対策:
  1. 通常は放っておけば自動的に更新される仕組みになっています（次回アクセス時に自動リロード）
  2. それでも古いままなら、ブラウザを完全終了 → 再起動 → 同じ URL を開き直し
  3. Mac Chrome で強制解決したい場合: DevTools → Application タブ → Storage → 「Clear site data」ボタン → リロード

生徒バディ「ルカ」が曖昧な返答しかしない

原因: 生徒に関する情報（苦手科目・講師コメント等）が登録されていない
対策: 管理画面で以下を登録してください:
  • 生徒編集 → 苦手科目を設定
  • スケジュール管理 → 今日以降の授業スロットを追加
  • 講師コメント記入時に「🤖 バディに共有する」を ON にして保存
これらが整うと、ルカが「黒木先生から〇〇って言われてるよ」と具体的に答えるようになります。

音声入力（🎤）を押しても無反応

対策: 第 7 章の「マイクが反応しない時」を参照。特に:
  • システム設定 → サウンド → 入力で、使いたいマイクが選ばれているか
  • Mac クラムシェル時は外部マイク必須
  • iPhone/iPad は非対応

申し送り Inbox に何も出てこない

原因: まだ誰も申し送りを書き込んでいない or 生徒バディがアラートを自動生成するような会話をしていない
確認方法: 生徒画面でルカに感情的な話（家庭・悩み）をしてみる。その後に教室長として /teacher の Inbox を確認 → alert が追加されているはず。

Claude API エラー（Anthropic 529）

症状: 会話がエラーで返ってこない
原因: Anthropic 側の一時的な過負荷（世界的に利用者が多い時間帯）
対策: 自動的に 3 回までリトライされます。それでもダメなら少し時間をおいてから再度話しかけてください。コード側の問題ではないので、待つ以外に解決策はありません。
